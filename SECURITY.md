# Security Policy

## Supported Versions

Only the latest release version receives security updates.

| Version | Supported          |
| ------- | ------------------ |
| < v0.1  | :x: No             |
| v0.1.x  | :white_check_mark: Yes |

---

## Reporting a Vulnerability

We take the security of FlientSec seriously. If you find a security vulnerability, please do not report it publicly via GitHub issues. Instead, report it privately to our security coordinators:

- **Contact Email:** `info.krishnasingh.codes@gmail.com`

Please include the following details in your report:
1. Type of issue (e.g. buffer overflow, SQL injection, auth bypass).
2. Steps to reproduce the vulnerability (including proof-of-concept scripts where possible).
3. The impact of the vulnerability on workstations or dashboards.

We will acknowledge receipt of your report within 48 hours and coordinate a patch release within 30 days. We request that you do not publish details of the vulnerability until a patch release is available.
